\chapter{Ausblick}

\begin{lernziele}
Nach diesem Kapitel kennen Sie sinnvolle nächste Schritte zur Vertiefung. Sie können einordnen, welche Themen nach diesem Einsteigerbuch folgen: vertiefte SysML-Modellierung, SysML v2, ROS2-Praxis, Simulation, Safety Engineering, Machine Learning und berufliche Anwendung.
\end{lernziele}

\section{Einleitung}
Dieses Buch bietet einen Einstieg. Es zeigt, wie SysML v2, MBSE und ROS2 am Beispiel eines autonomen Indoor-Roboters zusammenspielen. Nach dem Einstieg gibt es viele Wege zur Vertiefung. Dieses Kapitel ordnet die wichtigsten nächsten Schritte ein.

\section{SysML vertiefen}
Nach dem Einstieg lohnt es sich, SysML genauer zu lernen. Besonders relevant sind:

\begin{itemize}
    \item saubere Modellorganisation
    \item Wiederverwendung von Modellelementen
    \item Variantenmodellierung
    \item Schnittstellenkonzepte
    \item formale Anforderungen
    \item Parametrik und technische Analysen
    \item Modellreview und Modellqualität
\end{itemize}

\section{SysML v2}
SysML v2 modernisiert die Sprache und bringt eine präzisere textuelle und grafische Modellierung. Für Einsteiger ist SysML v1.x weiterhin verbreitet, aber SysML v2 wird langfristig wichtiger.

Wichtige Themen für später sind:

\begin{itemize}
    \item textuelle Modellierung
    \item präzisere Semantik
    \item bessere Werkzeugunterstützung
    \item Migration bestehender Modelle
    \item Verbindung zu modernen MBSE-Methoden
\end{itemize}

\section{ROS2 vertiefen}
Für Robotik sind folgende ROS2-Themen wichtige nächste Schritte:

\begin{itemize}
    \item Navigation2
    \item TF2
    \item Lifecycle Nodes
    \item Launch-Dateien
    \item Parameter
    \item rosbag
    \item Simulation
    \item Tests mit launch testing
\end{itemize}

\section{Simulation}
Gazebo oder andere Simulatoren ermöglichen Tests ohne echte Hardware. Das ist besonders wertvoll, wenn Algorithmen schnell ausprobiert werden sollen. Simulation kann helfen, Navigation, Sensorik, Hindernisse und Fehlerfälle früh zu prüfen.

\section{Safety Engineering}
Sobald Roboter mit Menschen in realen Umgebungen interagieren, werden Sicherheitsfragen wichtig. Dann reichen einfache Funktionsmodelle nicht mehr aus. Risikoanalysen, Sicherheitsanforderungen, FMEA, Gefährdungsanalysen und Normen müssen berücksichtigt werden.

\section{Machine Learning vertiefen}
ML-Integration endet nicht beim Laden eines Modells. Wichtige Vertiefungen sind:

\begin{itemize}
    \item Trainingsdaten
    \item Validierungsdatensätze
    \item Fehlklassifikationen
    \item Laufzeitoptimierung
    \item Edge-AI-Hardware
    \item Monitoring im Betrieb
    \item Umgang mit Unsicherheit
\end{itemize}

\section{Berufliche Relevanz}
Die Kombination aus Systems Engineering, SysML, ROS2 und KI ist in Robotik, Automotive, Industrieautomation und Forschung relevant. Wer diese Verbindung versteht, kann zwischen Architektur, Software und Tests vermitteln. Genau diese Schnittstellenkompetenz ist in vielen Projekten wertvoll.

\section{Vom Lernprojekt zum echten Projekt}
Der wichtigste Übergang nach diesem Buch ist der Schritt von einem didaktischen Modell zu einem echten Projekt. In einem echten Projekt werden Anforderungen widersprüchlicher, Schnittstellen technischer und Tests aufwendiger. Außerdem kommen organisatorische Themen hinzu: Versionierung, Reviews, Verantwortlichkeiten, Änderungsmanagement und Abnahme.

Das Lernprojekt liefert dafür eine Grundlage. Wer verstanden hat, wie eine Anforderung zur Architektur und weiter zu ROS2-Nodes und Tests führt, kann dieses Denken auf größere Systeme übertragen. Der Maßstab wächst, aber das Prinzip bleibt gleich: Problem verstehen, Modell aufbauen, Schnittstellen klären, Umsetzung ableiten und Nachweise planen.

\section{Zusammenfassung}
Der Einstieg ist geschafft, wenn Sie Anforderungen, Struktur, Verhalten, Schnittstellen, ROS2-Architektur und Tests zusammen denken können. Der nächste Schritt besteht darin, diese Konzepte in Werkzeugen und Code zu üben. MBSE wird erst durch Anwendung lebendig.
